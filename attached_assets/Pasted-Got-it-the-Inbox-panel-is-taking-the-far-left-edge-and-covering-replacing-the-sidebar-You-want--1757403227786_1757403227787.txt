Got it — the Inbox panel is taking the far-left edge and covering / replacing the sidebar. You want the icon-only sidebar to stay visible on the very left, and the Inbox to open next to it.

Here’s a drop-in layout fix that guarantees:

Sidebar is always rendered (64px when Inbox is open)

Inbox starts after the sidebar (not at left: 0)

Main content shifts accordingly

1) Use layout CSS variables + body attributes

Add these global rules (keep them at the end of your main CSS):

:root{
  --sb-w: 240px;           /* sidebar expanded */
  --sb-w-collapsed: 64px;  /* icon-only */
  --inbox-w: 360px;        /* inbox drawer width */
}

/* Sidebar: fixed and always visible */
.app-sidebar{
  position: fixed;
  inset: 0 auto 0 0;       /* top:0 right:auto bottom:0 left:0 */
  width: var(--sb-w);
  z-index: 40;              /* below inbox panel */
  transition: width .25s ease;
}
body[data-sidebar-collapsed="true"] .app-sidebar{
  width: var(--sb-w-collapsed);
}

/* Inbox: fixed panel that starts AFTER the sidebar */
.app-inbox{
  position: fixed;
  top: 0;
  left: var(--sb-w-collapsed);   /* <— key: reserve the icon bar */
  width: var(--inbox-w);
  height: 100vh;
  z-index: 50;                   /* above sidebar, below modals */
  transform: translateX(-110%);  /* hidden by default */
  transition: transform .25s ease, left .25s ease;
}
body[data-inbox-open="true"] .app-inbox{
  transform: translateX(0);      /* slide in */
}

/* Main content shifts based on what’s open */
.app-main{
  min-height: 100vh;
  margin-left: var(--sb-w);
  transition: margin-left .25s ease, padding-right .25s ease;
}
body[data-sidebar-collapsed="true"] .app-main{
  margin-left: var(--sb-w-collapsed);
}
body[data-inbox-open="true"] .app-main{
  /* leave room for the inbox panel on the left side */
  margin-left: calc(var(--sb-w-collapsed) + var(--inbox-w));
}

/* Optional: if you put Inbox on the RIGHT instead, swap to:
   .app-inbox { left:auto; right:0; }
   body[data-inbox-open="true"] .app-main { padding-right: var(--inbox-w); }
*/


If your theme already sets the sidebar to 240px, keep that; the crucial part is .app-inbox { left: var(--sb-w-collapsed) } and the main .app-main margins.

2) Ensure your components use these classes
// Sidebar.tsx
export function Sidebar() {
  return (
    <aside className="app-sidebar">
      {/* icon+labels … (labels auto hide when collapsed if you already do that) */}
    </aside>
  );
}

// InboxDrawer.tsx (left-side)
export function InboxDrawer({ open }: { open: boolean }) {
  React.useEffect(() => {
    document.body.toggleAttribute("data-inbox-open", open);
    if (open) {
      document.body.setAttribute("data-sidebar-collapsed", "true"); // lock icon-only
    } else {
      // restore happens elsewhere (your store), this is a safety
    }
  }, [open]);

  return (
    <aside aria-label="Inbox" className="app-inbox">
      {/* inbox content */}
    </aside>
  );
}

// AppLayout.tsx
export function AppLayout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <Sidebar />
      <InboxDrawer open={/* your state */ false} />
      <main className="app-main">{children}</main>
    </>
  );
}


If you’re already using the store I gave earlier, keep that—just make sure you set:

document.body.setAttribute("data-sidebar-collapsed","true") when Inbox opens

document.body.removeAttribute("data-inbox-open") when it closes

3) Common gotchas (quick checks)

Sidebar not visible? Make sure it’s not display:none anywhere on Inbox open. With the CSS above it remains fixed at left with width 64px.

Inbox still sits at far left? Confirm the element has the class app-inbox and that no other CSS sets left: 0 !important;.

Main content overlaps Inbox? Ensure your .app-main is not position: fixed and it uses the margin rules above; remove any older left-padding logic that fought this.

Apply those pieces and you’ll get: icon-only sidebar always visible on the far left, Inbox panel immediately to its right, and the dashboard shifting so nothing is covered.
