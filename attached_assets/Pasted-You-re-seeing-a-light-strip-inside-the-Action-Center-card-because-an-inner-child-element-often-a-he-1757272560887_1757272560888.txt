You’re seeing a light strip inside the Action Center card because an inner child element (often a header row or .card-body) still has a light background class (bg-white, bg-white/80, bg-base-100, bg-neutral-*, a DaisyUI card skin, etc.). Even if the outer card is correct, that inner bg will show exactly like your screenshot.

Here are two ways to fix it—pick one (both are safe and cheap to apply).

Option A — Minimal Tailwind fix (remove/neutralize the inner bg)

Open the Action Center component (e.g. src/components/family/ActionCenterCard.tsx).

Find the inner wrapper that holds the title/text (where “All caught up!” lives).
Remove any bg-* classes from it, or set it explicitly to bg-transparent.

Example before:

<div id="action-center-card" className="... p-4 ...">
  <div className="card-body bg-base-100 rounded-xl">   {/* <-- this causes the light strip */}
    <h3 className="card-title">Action Center</h3>
    <p>All caught up! 🎉</p>
  </div>
</div>


After:

<div id="action-center-card" className="relative isolate overflow-hidden rounded-2xl border border-white/8
                                       bg-gradient-to-b from-[#0b0b0e] to-[#0e0e14]
                                       shadow-[0_10px_28px_rgba(0,0,0,.45)]
                                       transition-all duration-200 p-4
                                       hover:border-[#D4AF37]/40
                                       hover:shadow-[0_0_0_1px_rgba(212,175,55,.35),0_18px_50px_rgba(212,175,55,.12)]">
  <div className="bg-transparent">  {/* NO bg-* classes here */}
    <h3 className="font-semibold">Action Center</h3>
    <p>All caught up! 🎉</p>
  </div>
</div>


Also remove any rounded-* on inner wrappers; keep rounding only on the outer card to avoid seams.

Option B — Guaranteed scoped override (one tiny CSS file)

If a UI library keeps injecting a bg, force it transparent only inside this card.

Create src/styles/action-center.css:

/* Make every known wrapper inside Action Center transparent */
#action-center-card :is(.card, .card-body, .card-title,
                        .bg-white, [class*="bg-white"],
                        .bg-base-100, [class*="bg-base-100"],
                        .bg-neutral, [class*="bg-neutral"]) {
  background: transparent !important;
}

/* Optional: links inherit theme and don't turn blue/underlined unless hover */
#action-center-card a { color: inherit; text-decoration: none; }
#action-center-card a:hover { text-decoration: underline; }


Import it and add an id on the outer card:

import "@/styles/action-center.css";

<div id="action-center-card" className="relative isolate overflow-hidden rounded-2xl border border-white/8
                                       bg-gradient-to-b from-[#0b0b0e] to-[#0e0e14]
                                       shadow-[0_10px_28px_rgba(0,0,0,.45)]
                                       transition-all duration-200 p-4
                                       hover:border-[#D4AF37]/40
                                       hover:shadow-[0_0_0_1px_rgba(212,175,55,.35),0_18px_50px_rgba(212,175,55,.12)]">
  {/* existing content (no need to touch inner classes now) */}
</div>


This guarantees no light bg survives inside that card.
