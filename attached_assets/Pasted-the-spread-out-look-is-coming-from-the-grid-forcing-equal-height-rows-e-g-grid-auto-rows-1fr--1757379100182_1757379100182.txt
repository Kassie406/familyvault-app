the “spread out” look is coming from the grid forcing equal-height rows (e.g., grid-auto-rows: 1fr) and/or cards set to h-full. That makes every row tall and creates those big belts of empty space.

Here are exact fixes so your dashboard looks like before, while still supporting Customize mode.

1) Make grid rows content-sized (not 1fr)

Replace the grid classes in SortableGrid:

Before

<div className="grid grid-cols-12 gap-4 auto-rows-[1fr] grid-auto-flow-dense items-stretch">


After

<div className="grid grid-cols-12 gap-4 auto-rows-auto [grid-auto-rows:minmax(0,_auto)] grid-auto-flow-dense items-start">


auto-rows-auto (and the arbitrary fallback) makes each row fit its content.

items-start prevents children from stretching vertically.

2) Don’t force cards to fill the cell height

In SortableCard:

Before

<div ref={setNodeRef} style={style} className={`${span} h-full`}>
  <div className="relative h-full">
    ...


After

<div ref={setNodeRef} style={style} className={`${span}`}>
  <div className="relative h-auto">
    ...


If you have a reusable Card wrapper you set to h-full, change it to h-auto for dashboard cards.

3) Remove any min-heights that inflate rows

Search your cards for things like min-h-[400px] or min-h-screen and either remove them or cap them:

<div className="min-h-0 h-auto max-h-none"> ... </div>

4) Keep the “old look” in view mode (optional)

If you want to be extra safe, only apply the DnD wrapper while editing:

{editing ? (
  <SortableGrid ... />   // uses the grid classes above
) : (
  <div className="grid grid-cols-12 gap-4 auto-rows-auto [grid-auto-rows:minmax(0,_auto)]">
    {visible.map(({ id }) => renderCard(id))}
  </div>
)}

5) Tiny global safeguard (optional)
/* Prevent trailing spacing inside cards from expanding rows */
.dashboard .card > *:last-child { margin-bottom: 0; }

Quick checklist

 Grid uses auto-rows-auto (not 1fr)

 No h-full on the grid item wrapper or card root

 No large min-h-* on cards

 items-start on the grid

Apply those three changes and your dashboard will snap back to the tight layout you had, while “Customize” still lets you drag, hide, and reorder. If you paste your current SortableGrid.tsx (just the JSX line), I’ll return it edited line-for-line.
