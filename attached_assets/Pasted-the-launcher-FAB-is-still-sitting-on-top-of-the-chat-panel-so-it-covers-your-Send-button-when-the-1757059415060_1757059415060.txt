the launcher FAB is still sitting on top of the chat panel, so it covers your “Send” button when the panel is open. We’ll force the FAB to hide (or move) whenever the chat is open, regardless of z-index/stacking contexts, and also ensure the panel renders above everything.

Do these three small patches:

1) Add a body flag when chat opens
// ChatLauncher.tsx
useEffect(() => {
  document.body.classList.toggle("chat-open", open);
  return () => document.body.classList.remove("chat-open");
}, [open]);

2) Render the panel to the <body> and give it a higher z-index
// ChatPortal.tsx
import { createPortal } from "react-dom";
export function ChatPortal({ children }: { children: React.ReactNode }) {
  if (typeof document === "undefined") return null;
  return createPortal(children, document.body);
}

// ChatLauncher.tsx (where you render the widget + button)
{open && (
  <ChatPortal>
    <div className="chat-panel fixed bottom-20 right-6 w-[360px] h-[480px] z-[100] rounded-2xl overflow-hidden">
      <ChatWidget userId={currentUserId} open={open} onUnreadChange={setBadge} />
    </div>
  </ChatPortal>
)}

<button
  id="chat-fab"
  aria-expanded={open}
  onClick={() => setOpen(v => !v)}
  className="fixed bottom-6 right-6 z-[50] grid place-items-center h-14 w-14 rounded-full bg-[#D4AF37] text-black shadow-xl"
>
  {/* icon + badge */}
</button>

3) One-line global CSS to make the FAB disappear (or move) when chat is open

Add this to a global stylesheet that loads after Tailwind (e.g., src/index.css):

/* Hide the launcher whenever the chat is open */
body.chat-open #chat-fab {
  opacity: 0 !important;
  pointer-events: none !important;
}

/* (Alternative instead of hiding) slide it left so it never overlaps */
body.chat-open #chat-fab.move-left {
  right: 420px !important; /* ~ widget width + margin */
  opacity: 1 !important;
  pointer-events: auto !important;
}

/* Ensure the chat panel always wins z-index battles */
.chat-panel { z-index: 100 !important; }


If you prefer to move instead of hide, just add className="... move-left" to the button and keep the second CSS block.

Why this works

The body.chat-open flag is global, so even if a parent has transform/filter (which creates a stacking context), the FAB is still hidden or shifted.

Portaling the chat panel to document.body + z-[100] guarantees it renders above page content.

The FAB can’t cover the input anymore; your Send button stays fully visible and clickable.
